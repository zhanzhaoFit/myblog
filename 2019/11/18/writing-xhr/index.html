# 同源策略定义

由netscape提出的一个安全策略，是浏览器最核心也最基本的安全功能，同源是指域名、协议、端口相同。

# 受同源策略限制的行为

  • cookie、localStoage和indexDB的读取
  • DOM节点的获取
  • XMLHttpRequest请求

# XMLHttpRequest请求

# 代码实现

//IE7及以上版本和其他浏览器原生支持XHR对象
function createXHR() {
	if (typeof XMLHttpRequest !== "undefined") {
		return new XMLHttpRequest();
	} else {
		throw new Error("No Available XHR object");
	}
}
var xhrObj = createXHR();
xhrObj.onreadystatechange = function() {
	if (xhrObj.readyState == 4) {
		try { //如果在超时终止后访问status属性会导致报错，可以使用try...catch...来避免报错
			if ((xhrObj.status >= 200 && xhrObj.status < 300) || xhrObj.status == 304) {
				//请求成功数据处理
				console.log(xhrObj.responseText);
				var myHeader = xhrObj.getResponseHeader("myHeader"); //获取相应的响应头信息
				var allHeaders = xhrObj.getAllResponseHeaders(); //获取包含所有头信息的长字符串
			} else {
				throw new Error("No Available Response with status:" + xhrObj.status);
			}
		} catch (ex) {
			//假设由ontimeout事件处理程序处理
		}
	}
}
/* Firefox引入load事件用以替代onreadystatechange事件兵器也不用检查readyState属性 */
xhrObj.onload = function(event) {
	try { //如果在超时终止后访问status属性会导致报错，可以使用try...catch...来避免报错
		if ((xhrObj.status >= 200 && xhrObj.status < 300) || xhrObj.status == 304) {
			//请求成功数据处理
			console.log(xhrObj.responseText);
			var myHeader = xhrObj.getResponseHeader("myHeader"); //获取相应的响应头信息
			var allHeaders = xhrObj.getAllResponseHeaders(); //获取包含所有头信息的长字符串
		} else {
			throw new Error("No Available Response with status:" + xhrObj.status);
		}
	} catch (ex) {
		//假设由ontimeout事件处理程序处理
	}
}
/* Mozilla引入了progress事件，这个事件会在浏览器接收新数据期间周期性触发,通过这些信息可以创建进度指示器 */
xhrObj.onprogress = function(event) {
	/*
	 *event对象提供lengthComputable、position、totalSize三个属性
	 *lengthComputable：表示进度信息是否可用的布尔值
	 *position：表示已经接收的字节数
	 *totalSize：表示根据Content-Length响应头部确定的预期字节数
	 */
	if (event.lengthComputable) {
		console.log("Received " + event.position + " of " + event.totalSize + " bytes");
	}
}
xhrObj.open("get", "exampleUrl", true); //开通一个XHR请求
xhrObj.setRequestHeader("myHeader", "myValue"); //设置自定义请求头信息
/* XMLHttpRequest2级 */
xhrObj.timeout = 1000; //设置超时时间（单位毫秒）
xhrObj.ontimeout = function() { //监听超时回调
	throw new Error("Request did not return in the limit time")
}
xhrObj.overrideMimeType("text/xml"); //重写服务器返回的MIME类型
//支持overrideMimeType()方法浏览器：Firefox,Safari4+,Opera10.5和Chrome
xhrObj.send(new FormData(formDom)); //XHR对象能够识别传入数据类型为FormData的实例，并配置适当的头信息，而不必明确地在XHR对象上设置请求头部
//支持formdata的浏览器：Firefox4+,Safari5+,Chrome,Android3+版WebKit
/* XMLHttpRequest2级 */
xhrObj.send(null); //执行发送，没有数据内容传null
/*
 *默认发送XHR请求同时会发送下列头部信息
 *Accept：浏览器能够处理的内容类型
 *Accept-Charset：浏览器能够显示的字符集类型
 *Accept-Encoding：浏览器能够处理的压缩编码类型
 *Accept-Language：浏览器当前设置的语言类型
 *Connection：浏览器与服务器之间连接的类型
 *Cookie：但钱页面设置的任何Cookie
 *Host：发出请求的页面所在的域
 *Referrer：发出请求多的页面地址
 *User-Agent：浏览器用户代理字符串
 */

# 跨域处理办法

  • JSONP（只能发送）
  • CORS（跨域资源分享），是跨域AJAX请求的根本解决办法，允许任何类型的请求
  • websocket（是一种通信协议，不受同源政策影响，只要服务器支持就可以进行跨域通信）

# JSONP（JSON With Padding）

浏览器的同源策略只是阻止了通过AJAX技术跨域获取资源，并没有禁止跨域获取资源（图片、css、js等）的获取；JSONP就是根据这个原理实现的跨域资源的获取；

# 支持兼容性

浏览器终端支持性好，可以向不支持CROS的网站请求数据。

# 实现

  • 识别请求的URL，提取callback参数的值，并动态生成一个执行该参数值得javascript语句；
  • 将需要返回的数据放入动态生成的函数中，等待其加载到页面被执行。

# 存在的缺陷

  • 无法发送post请求，也就是说JSONP技术只能用于异域资源的获取，无法上传或修改异域数据；
  • 无法检测JSONP请求是否失败；
  • 可能存在的安全隐患（无法避免服务端在返回的跨域资源中添加恶意代码），因此使用JSONP技术时要确保服务器的可信度。

# CROS（Cross-origin Resource Sharing）

它允许浏览器向跨域服务器发送XMLHttpRequest请求，从而克服了AJAX只能同源使用的限制。对于开发者而言CROS通信与同源的AJAX通信没有差别，浏览器一旦发现AJAX请求跨域，就会自动添加一些附加的头信息，有时会多出一次附加的请求，单用户无感知。

# 应用场景

  • 有XMLHttpRequest或Fetch发起的跨域HTTP请求；
  • web字体（CSS中通过@font-face使用跨域字体资源）；（因此网站就可以发布TrueType字体资源，并只允许已授权网站进行跨站调用）；
  • webGL贴图

# 兼容性

需要浏览器和服务器同时支持；目前IE10以上版本及其他主流浏览器均支持。浏览器将CROS请求分为简单请求和非简单请求；

# 其他跨域技术

# 图像ping

是与服务器进行简单、单向的跨域通信的一种方式，请求数据是通过查询字符串形式发送的，浏览器得不到任何具体的数据；但是通过load和error事件可以知道响应是什么时候接收到的。

# 应用场景

在线广告跟踪浏览量的主要方式。

# 实现方式

var imgData = new Image();
imgData.onload = imgData.onerror = function() {
	console.log("done!";)
}
imgData.src = "exampleUrl?key=value&key1=value1"

# Comet

一种服务器向页面推送数据的技术；能够让信息近乎实时的被推送到页面上。

# 应用场景

体育比赛比分、股票报价；

# 实现方式

  • 长轮询：由页面发起一个到服务器的请求，然后服务器一直保持链接打开，直到有数据可发送；发送完数据之后，浏览器关闭连接，随机又发起一个到服务器的新请求；这一过程在页面打开期间一直持续不断。
  • HTTP流：在页面整个生命周期内，只使用一个HTTP链接。具体来说就是浏览器向服务器发送一个请求，而服务器保持链接打开，然后周期性地向浏览器发送数据。 在Firefox、Safari、Opera、Chrome中可以在readystatechange事件中检测readyState值是否为3，就可以利用XHR对象实现HTTP流。随着不断从服务器结束数据，readyState的值会周期性地变为3；当readyState值变为3是，responseText属性中会保存接收到的所有数据，因此需要判断从什么位置开始是最新数据。
function createHttpStreamXHR(url, progress, finished) {
	var xhrObj = new XMLHttpRequest(),
		receivedIndex = 0;
	xhrObj.open("get", url, true);
	xhrObj.onreadystatechange = function() {
		var result = "";
		if (xhrObj.readyState === 3) {
			result = xhrObj.responseText.substring(receivedIndex);
			receivedIndex += result.length;
			progress(result);
		} else if (xhrObj.readyState === 4) {
			finished(xhr.responseText);
		}
	}
	xhrObj.send(null);
	return xhrObj;
}

# SSE(Server-Sent Events)

服务器发送事件，是围绕只读Comet交互推出的API或模式；用于创建到服务器的单向连接，服务器可以通过这个连接发送任意数量的数据。服务器响应的MIME类型必须是text/event-stream，而且是浏览器中JavaScriptAPI能解析格式输出；SSE支持短轮询、长轮询和HTTP流，而且能再断开连接时自动确定何时重连。

# 兼容性

支持SSE的浏览器有Firefox6+、Safari5+、Opera11+、Chrome和IOS4+版Safari；

# SSE API

预定新的事件流，首先创建一个EventSource对象，并传入一个入口点(注意入口点URL必须和所在页面同源)

/*
 *首先创建EventSource实例
 *open:在建立的时候触发
 *message:在从服务器接收到新事件时触发
 *error:在无法建立连接时触发
 *close:可以直接强制断开连接
 */
var source = new EventSource("myevents.php");
source.onopen = function() {
	//建立时处理
}
source.onmessage = function(event) {
	/*
	 *服务器时间会通过一个持久的http响应发送，响应的MIME类型为text/event-stream
	 *响应格式为纯文本，最简单的情况是每个数据项都带有"data:"前缀
	 *
	 *data: bar
	 *
	 *data: foo
	 *
	 *对于单行数据后有空行的，event.data可以直接获取到数据内容
	 *
	 *data: bar
	 *data: foo
	 *
	 *对于多个连续的以"data:"开头的数据行，将作为多段数据解析，每个值之间以一个换行符分隔
	 *因此服务器生成事件流时不能忘了多添加一行。
	 *
	 *可以通过id:前缀给特定事件关联一个ID，ID位于行前或行后都行
	 *data: foo
	 *id: 1
	 *
	 *设置ID后，EventSource会跟踪上一次触发的事件。
	 *如果连接断开，会向服务器发送一个名为Last-Event-ID的特殊HTTP头部的请求，以便服务器知道下次该出发那个事件
	 */
	var data = event.data;
	//接收到新事件处理
}
source.onerror = function() {
	//无法建立处理
}
source.close();
